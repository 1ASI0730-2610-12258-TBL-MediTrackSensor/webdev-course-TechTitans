## Errores comunes que todos cometemos

"Primero, hablemos de algo completamente normal: los errores."

"Incluso los desarrolladores profesionales cometen errores todos los días. La diferencia es que aprenden a encontrarlos y corregirlos."

Algunos errores muy comunes son:

* Olvidar cerrar una etiqueta HTML.
* Escribir mal el nombre de una etiqueta.
* Olvidar guardar los cambios antes de probar la página.
* Confundir HTML con CSS.
* Borrar accidentalmente una parte del código.

> **Nota visual:** Mostrar un ejemplo de una etiqueta sin cerrar y luego la versión corregida.

"Si algo no funciona, no significa que hayas hecho algo mal para siempre. Normalmente significa que hay un pequeño detalle que necesita corregirse."

"Aprender a buscar errores es una de las habilidades más importantes de cualquier programador."

## Buenas prácticas para escribir mejor código

"Ahora veamos algunos hábitos que te ayudarán a mantener tu código limpio y fácil de entender."

### Mantén tu código ordenado

"Usa espacios e indentación para organizar tus etiquetas."

> **Nota visual:** Comparar código desordenado con código correctamente indentado.

"El código ordenado es más fácil de leer y corregir."

### Usa nombres claros

"Cuando agregues clases o identifiques elementos, utiliza nombres que describan su propósito."

Por ejemplo:

* Mejor: `titulo-principal`
* Menos claro: `cosa1`

"Tu yo del futuro te lo agradecerá."

### Haz cambios pequeños

"Cuando estés experimentando, realiza un cambio a la vez."

"Así será más fácil descubrir qué causó un error si algo deja de funcionar."

### Prueba constantemente

"No esperes hasta terminar toda la página para revisarla."

"Haz cambios pequeños y observa el resultado frecuentemente."
